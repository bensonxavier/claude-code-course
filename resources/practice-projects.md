# Practice Projects and Success Criteria

These project ideas are meant to make the course feel less like a checklist and more like real skill-building.

## Suggested mini-projects

### 1. Personal notes assistant
Create a small folder with notes and ask Claude to summarize, categorize, or rewrite them. Success means the output is useful enough that you would keep it.

### 2. Small code helper
Ask Claude to write, explain, or refactor a tiny script. Success means the result is correct, readable, and clearly explained.

### 3. CLAUDE.md refinement project
Take an existing project and improve its instructions. Success means Claude's responses become more contextual and aligned with the project.

### 4. Planning and loop exercise
Give Claude a multi-step task and review whether the plan or loop improved the final result. Success means the task was completed with fewer mistakes after refinement.

### 5. Git review workflow
Use Claude for commit message suggestions or PR summaries. Success means the output is concise, accurate, and useful in practice.

## Reflection questions

After each project, ask:
- What did Claude do well?
- Where did it fail or become vague?
- What prompt, context, or constraint would improve the next attempt?

These questions help the course focus on judgment, iteration, and real-world usefulness rather than simple completion.
