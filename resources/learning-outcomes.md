# Learning Outcomes and Assessment

This document defines the intended outcomes for each module and the minimum evidence that shows a learner has actually built the skill.

## Module 1-2: Claude CLI

### Learning outcomes
- Install and verify Claude Code.
- Run one-shot prompts and interactive sessions.
- Use file context effectively.
- Inspect and refine output when it is incomplete or inaccurate.

### Assessment check
A learner has succeeded when they can:
- start Claude Code from the terminal,
- complete a task with a clear prompt and file context,
- and improve the result after one round of refinement.

## Module 3-4: CLAUDE.md

### Learning outcomes
- Create a useful project-level CLAUDE.md file.
- Add context, constraints, and workflow guidance.
- Observe how Claude responds differently when the instructions are clearer.

### Assessment check
A learner has succeeded when they can:
- create a CLAUDE.md file for a real or sample project,
- explain why the file improved the outcome,
- and identify at least one instruction that Claude ignored or misunderstood.

## Module 5-6: Plan and Loop Modes

### Learning outcomes
- Use planning mode for multi-step work.
- Review and adjust a proposed plan before execution.
- Use loop-based workflows to improve outcomes over iterations.

### Assessment check
A learner has succeeded when they can:
- run a realistic task in plan mode,
- explain what changed after review or iteration,
- and complete the task with fewer errors than the first attempt.

## Module 7-8: MCP Servers

### Learning outcomes
- Understand the role of MCP in connecting tools to an agent.
- Configure a simple MCP server.
- Use a tool through an agent workflow.

### Assessment check
A learner has succeeded when they can:
- configure a basic server,
- trigger a tool call from a prompt,
- and describe the value of the tool in plain language.

## Module 9-10: Parallel Subagents

### Learning outcomes
- Break a task into parallel workstreams.
- Coordinate work across multiple agents or sub-tasks.
- Merge results into a coherent final outcome.

### Assessment check
A learner has succeeded when they can:
- split a task into independent parts,
- compare the parallel result with a single-agent result,
- and explain where coordination mattered.

## Module 11-12: Git Integration

### Learning outcomes
- Use Claude Code in a git-based workflow.
- Create useful commit messages, PR summaries, and changelog notes.
- Help resolve simple merge conflicts or review diffs.

### Assessment check
A learner has succeeded when they can:
- review changes with Claude in a repository,
- produce a useful commit or PR summary,
- and explain how the assistance improved the workflow.

## General assessment principle

The learner should not be considered finished with a module just because the checklist is complete. The stronger evidence is that they can:
- complete a small realistic task,
- reflect on what worked and what failed,
- and improve the outcome through iteration.
