# Task 1: Create and test a CLAUDE.md file

## Objective
Learn how to give Claude Code project-specific instructions.

## Steps
1. Create a small test project folder with one or two sample files.
2. Add a CLAUDE.md file with sections for project description, tech stack, folder structure, and rules.
3. Start Claude Code inside that folder and ask: "What context do you have about this project?"
4. Add a rule such as a coding style preference or workflow constraint.
5. Ask Claude to do a small task and observe whether the rule is reflected in the response.
6. Refine the CLAUDE.md content based on what was useful or ignored.

## Deliverable
A polished CLAUDE.md file plus a note in [resources/my-notes.md](../../resources/my-notes.md) about what made the context effective.

## Success criteria

You have completed the task well when:
- the CLAUDE.md file clearly describes the project,
- the instructions are specific enough to guide Claude,
- and the resulting output is more useful than it would have been without the file.

## Reflection loop

If Claude ignores part of your instructions, ask yourself:
- was the rule too vague,
- was the context missing,
- or did the task need a more concrete prompt?
